{% blocktranslate %}Aquí se cargan y consultan datos maestros (pólizas y siniestros) con usuario y contraseña, y se filtran y buscan casos ya registrados. En el sector asegurador, los cores operativos habituales incluyen soluciones como Acsel/x (Consis) o Rector (Indra); este panel no sustituye ese core completo, pero sí el paralelo en Excel/Access que muchas áreas usan para cuadros y seguimiento. {% translate "Qué datos cargamos y cómo se relacionan" %}

{% blocktranslate %}En las dos ilustraciones siguientes se ve, en lenguaje sencillo, qué guardamos en el sistema: primero la cartera propia (pólizas y siniestros) que usted carga desde aquí; después las cifras de mercado de referencia publicadas por la SUDEASEG, que sirven para contextualizar resultados frente al sector (no reemplazan informes oficiales ni dictámenes regulatorios).{% endblocktranslate %}

{% translate "Pólizas y siniestros (operativo)" %}

{% blocktranslate %}Cada archivo subido queda registrado como un lote de carga (fecha, resumen y trazabilidad). Las pólizas se guardan como fichas con un identificador único de póliza, cohorte, prima y estado. Los siniestros se enlazan siempre a una póliza que ya exista en el sistema, para poder medir siniestralidad sobre la misma cartera que usted cargó.{% endblocktranslate %}

{% translate "Mercado SUDEASEG (referencia)" %}

{% blocktranslate %}Aquí se almacenan series por empresa y por tiempo (por ejemplo año y mes), con nombres homogéneos para poder aislar a La Fe y comparar con el conjunto del mercado. Esos datos no se «enganchan» fila a fila a sus pólizas: son una referencia externa para tableros e indicadores (portal y API de cómputo) que contrastan su cartera con el sector.{% endblocktranslate %}
